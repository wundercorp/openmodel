# Governance

baseui.sh uses a maintainer-led, contribution-driven governance model.

## Maintainers

Maintainers review contributions, manage releases, enforce project standards, and approve changes to the public API and design invariants.

## Decisions

Routine decisions are made through pull-request review. Significant changes require an RFC issue covering motivation, API, accessibility, migration, alternatives, and release impact.

## Becoming a maintainer

Consistent contributors may be invited after demonstrating sound technical judgment, respectful collaboration, and sustained ownership of the project.
